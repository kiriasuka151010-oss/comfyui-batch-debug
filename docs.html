批量参数扫描 · 多 LoRA 链式扫描 · 交互评分查看器

v2.1 6 槽 LoRA D站 Pipeline HTML Viewer

一句话说清楚

  • 做什么：把 AI 绘图调参从"手动改参数 → 点 Queue → 记笔记"变成"配置一次 → 批量扫描 → 打分筛选"
  • 解决什么：LoRA 该用多大权重？CFG 调到几效果好？不再靠猜，用数据说话
  • 怎么用：D站选参考图 → 配 LoRA 权重范围 + CFG 范围 → 点 Queue → 打开 viewer 打分

一、工作流程

🖼 D站选图
AnimaDex + Danbooru → 🏷 Tag 反推
AnimaPromptConverter → ⚙ 配置扫描
6槽LoRA + CFG → 🚀 批量执行
笛卡尔积 → 📦 结构化输出
sweep 文件夹 → ⭐ 评分筛选
viewer.html

不需要手写 prompt。D 站选图 → Tag 自动反推 → 参数扫描 → 出图 → 打分，全链路。

二、三个节点

Config BatchDebugConfig

6 个 LoRA 槽，每个独立：下拉选模型 / 开关 ON-OFF / 权重范围。配合 CFG 和 Steps 扫描范围，所有参数做笛卡尔积。Seed 固定不变，确保公平对比。

参数类型说明
prompts多行留空即可——D 站 pipeline 接管 prompt
lora_N_enabled开关ON 才参与扫描（默认 OFF）
lora_N_name下拉自动列出 ComfyUI loras 目录下所有 .safetensors 文件
lora_N_weight_min / max / stepsFloat / Int权重扫描范围（如 0.3→1.0×3 → [0.3, 0.65, 1.0]）
lora_weight_clipFloat全局 CLIP 倍率，对所有 LoRA 的 clip weight 生效
cfg_min / max / stepsFloat / IntCFG 扫描范围
seedInt固定 seed，所有组合共用
steps_min / max / countInt采样步数扫描范围
组合数 = LoRA 槽权重组合 × CFG 数 × Steps 数。超过 200 张会打印内存警告。

Execute BatchDebugExecute

接收上游 MODEL / CLIP / VAE / CONDITIONING / LATENT，遍历所有参数组合。链式叠加多 LoRA → 编码/直通 conditioning → KSampler → VAE 解码。

参数说明
model / clip / vae从 UNETLoader / CLIPLoader / VAELoader 接入（Anima 模型）
positive / negative从 CLIPTextEncode 接入。conditioning 模式下直通，不重新编码
latent_image从 EmptyLatentImage 接入，控制出图分辨率
config_json从 BatchDebugConfig 接入
prompt_sourceconditioning：上游 CLIPTextEncode 直通（推荐）；reencode：节点内重新编码
sampler_name / scheduler采样器 & 调度器，从 ComfyUI 实时拉取列表
denoise降噪强度，文生图保持 1.0
save_preview每轮实时存预览图到 output 目录
preview_prefix预览图输出子目录
conditioning 模式：AnimaPromptConverter → CLIPTextEncode 的 conditioning 直通 KSampler，保证画质与工作流一致。Config 里的 prompt 仅用于元数据标注。

GridSave BatchDebugGridSave

接收 Execute 的图片批次 → 输出结构化 sweep 文件夹。

参数说明
images / metadata_json从 Execute 的输出接入
columns / cell_width拼图列数 / 格宽，标签自动截断
max_grid_dimension拼图最大尺寸，超出自适应缩放
label_font_size / label_format标注字体大小 & 模板（变量: {lora_info} {cfg} {steps}）
filename_prefix输出子目录，支持 %date% %time% 占位符
save_individual / save_grid / save_metadata_csv独立开关：原图 / 拼图 / CSV

三、输出结构

每次扫描生成一个独立文件夹：

sweep_20260621_163000/
├── img_0000.png ← 每张原图，序号与 CSV 行对应
├── img_0001.png
├── ...
├── grid.png ← 标注拼图预览
├── metadata.csv ← 参数表
├── report.json ← viewer 数据源
└── viewer.html ← 双击打开，拖入 report.json 即可评分筛选

四、评分查看器

viewer.html 是独立可复用工具——所有扫描共用一个：

功能操作
⭐ 评分点击卡片下方星星打分（1-5），存在浏览器本地
🔍 筛选按 CFG / Steps / LoRA 下拉筛选，或搜索关键词
📊 排序按序号 / CFG / Steps / 评分排序
🖼 放大双击图片 lightbox 全屏
📥 导出一键导出带评分的 CSV
📂 加载拖入 report.json 即加载（不受浏览器安全策略限制）

五、技术要点

特性实现
多 LoRA 链式load_lora_for_models 逐层叠加，模型自动 clone 不污染原始引用
Conditioning 直通上游 CLIPTextEncode 的 conditioning 直接传给 KSampler——保证 D 站 tag 拼装质量
去重保护min==max 时自动去重，不生成重复参数组合
中断安全try/finally 包裹 + LoRA 缓存清理，Ctrl+C 不泄漏 GPU 内存
零额外依赖仅使用 ComfyUI 内置 torch / numpy / PIL
