v2.1 6 槽 LoRA D站 Pipeline HTML Viewer

一句话说清楚

  • 做什么：把 AI 绘图调参从"手动改→点 Queue→记笔记"变成"配置一次→批量扫描→HTML 打分筛选"
  • 解决什么：LoRA 该用多大权重？CFG 调到几效果好？不再靠猜，用数据说话
  • 怎么用：D站选参考图 → 配 LoRA 权重范围 → 点 Queue → 打开 viewer 打分

一、工作流程

🖼 D站选图
AnimaDex + Danbooru → 🏷 Tag 反推
AnimaPromptConverter → ⚙ 配置扫描
6槽LoRA + CFG → 🚀 批量执行
笛卡尔积 → 📦 结构化输出
sweep文件夹 → ⭐ 评分筛选
viewer.html

不需要手写 prompt。D站选图 → Tag 自动反推 → 参数扫描 → 出图 → HTML 打分，全链路。

二、输出结构

每次扫描生成一个独立文件夹：

sweep_20260621_163000/
├── img_0000.png ← 每张原图，序号与参数对应
├── img_0001.png
├── ...
├── grid.png ← 拼图预览
├── metadata.csv ← 参数表
├── report.json ← viewer 数据源
└── viewer.html ← 双击打开，拖入 report.json 即可评分

三、三个节点

Config BatchDebugConfig

6 个 LoRA 槽，每个独立：下拉选模型 / 开关 / 权重范围(min→max×步数)。配合 CFG 和 Steps 扫描范围，所有参数做笛卡尔积。

参数类型说明
prompts多行留空即可——D站 pipeline 接管
lora_N_enabled开关ON 才参与扫描
lora_N_name下拉自动列出 ComfyUI loras 目录所有文件
lora_N_weight_min/max/stepsFloat/Int权重扫描范围
cfg/steps/seedFloat/Int扫描范围 + 固定 seed（公平对比）

Execute BatchDebugExecute

接收上游 MODEL/CLIP/VAE/CONDITIONING/LATENT，遍历所有参数组合。链式叠加多 LoRA → 编码/直通 conditioning → KSampler → VAE 解码。

参数说明
prompt_sourceconditioning：上游 CLIPTextEncode 直通（推荐，支持 AnimaPromptConverter）；reencode：节点内重新编码
save_preview每轮实时存预览图到 output 目录
sampler/scheduler/denoise采样器配置
conditioning 模式：用户接 AnimaPromptConverter → CLIPTextEncode 的 conditioning，Execute 不重新编码 prompt，保证画质与工作流一致。

GridSave BatchDebugGridSave

接收图片批次 → 输出结构化 sweep 文件夹（单张图 + 拼图 + CSV + report.json + viewer.html）。

参数说明
columns/cell_width拼图列数/格宽，标签自动截断适配
label_format标注模板，变量：{lora_info} {cfg} {steps}
filename_prefix输出子目录，支持日期占位符
save_*独立开关：原图/拼图/CSV

四、评分查看器

viewer.html 是独立可复用工具——扫 100 次也只用这一个：

功能操作
⭐ 评分点击卡片下方星星打分（1-5），评分存在浏览器本地
🔍 筛选按 CFG / Steps / LoRA 下拉筛选，或搜索关键词
📊 排序按序号/CFG/Steps/评分排序
🖼 放大双击图片 lightbox 全屏查看
📥 导出一键导出带评分的 CSV，可进 Excel 继续分析
📂 加载拖入 report.json 即加载（浏览器安全策略不影响拖拽）
使用：打开 sweep 文件夹 → 双击 viewer.html → 拖入同目录的 report.json → 开始打分。

五、D站 Pipeline 工作流

AnimaDexBrowser→ DanbooruBrowser→ AnimaPromptConverter→ CLIPTextEncode→ BatchDebugExecute→ GridSave

选 artist → 选图 → Tag 反推 → 自动化扫描。拖入 d站_loRA_对比_workflow.json 即可开始。

六、技术要点

特性实现
多 LoRA 链式load_lora_for_models 逐层叠加，模型自动 clone 不污染原始引用
Conditioning 直通上游 CLIPTextEncode 的 conditioning 直接传给 KSampler，不重新编码——保证 D站 tag 拼装质量
去重保护min==max 时自动去重，不会生成重复参数组合
中断安全try/finally 包裹循环 + LoRA 缓存清理，Ctrl+C 不会泄漏 GPU 内存
零依赖仅使用 ComfyUI 内置 torch/numpy/PIL，无需额外 pip install
