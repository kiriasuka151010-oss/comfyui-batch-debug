一、概览

当你需要回答这些问题时，这个插件就是答案：

  • "CFG 调到几最好看？"
  • "这个 LoRA 权重 0.4 和 0.8 差多少？"
  • "同一个 prompt，不同 seed 有几张能用的？"
  • "Steps 从 20 到 40，画质提升值不值得等？"

三个节点，笛卡尔积扫描所有参数组合，自动出图 + 网格对比 + CSV 元数据，省去你手动改参数点 Queue 的重复劳动。

二、节点详解

Config BatchDebugConfig — 参数扫描配置

设置你要扫描的参数范围和组合。所有参数做笛卡尔积，输出一个 JSON 配置字符串传给 Execute。

参数类型说明
prompts 多行文本 每行一个 prompt。留空即可（conditioning 模式下由 D站 pipeline 接管 prompt）。
lora_sweep 多行文本 多 LoRA 扫描。每行一个 LoRA：文件名 最小值 最大值 步数 [clip_mult]
例：anima线稿画风.safetensors 0.3 1.0 4
#开头为注释行。多个 LoRA 做笛卡尔积。
lora_weight_clipFloat全局 CLIP 强度倍率，对所有 LoRA 的 clip weight 生效。
cfg_min / max / stepsFloat / IntCFG 扫描范围。如 3→9 分 4 步 → [3, 5, 7, 9]
seedInt固定 seed（不扫描！）。所有组合用同一个 seed，确保公平对比。
steps_min / max / countInt采样步数扫描。
💡 多 LoRA 示例：
anima线稿画风.safetensors 0.3 1.0 3 → 权重 [0.3, 0.65, 1.0]
anima美学增强.safetensors 0.5 1.5 2 → 权重 [0.5, 1.5]
组合数：3 × 2 × CFG数 × Steps数 = 总出图数

Execute BatchDebugExecute — 批量执行引擎

接收上游的 MODEL / CLIP / VAE / CONDITIONING / LATENT，遍历所有参数组合，每轮跑 KSampler + VAE 解码，输出所有图片 + 元数据 JSON。

参数类型说明
model / clip / vaeMODEL / CLIP / VAE从 UNETLoader / CLIPLoader / VAELoader 接入。
positive / negativeCONDITIONING从 CLIPTextEncode 接入。在 conditioning 模式下直通使用，不重新编码。
latent_imageLATENT从 EmptyLatentImage 接入，控制出图分辨率。
config_jsonSTRING从 BatchDebugConfig 接入。
prompt_source下拉关键参数，见下方详解。
sampler_name下拉采样器（euler / dpmpp_2m / …）。从 ComfyUI 实时拉取列表。
scheduler下拉调度器（normal / karras / sgm_uniform / …）。
denoiseFloat降噪强度，图生图时使用。纯文生图保持 1.0。

🔑 prompt_source 两种模式

✨ conditioning（推荐）

使用上游 CLIPTextEncode 的 conditioning，不重新编码 prompt。

这样你可以把 AnimaPromptConverter → CLIPTextEncode 接进来，prompt 质量 = 工作流原生质量。

Config 里的 prompt 文本仅用于元数据标注和文件名。

📝 reencode

每轮用 Config 的 prompt 文本重新调用 clip.tokenize() → encode_from_tokens_scheduled()。

适合不需要 AnimaPromptConverter 的场景（如英文 prompt 直出），或需要每轮动态改 prompt 文本。

⚠ 中断安全：循环中用 try/finally 包裹，LoRA 缓存在用户中断或异常时自动释放，不会泄漏 GPU 内存。

GridSave BatchDebugGridSave — 结果保存（输出节点）

接收 Execute 的图片批次和元数据，输出三样东西：单张图、标注网格图、CSV 评分表。

参数类型说明
imagesIMAGE从 Execute 的 images 输出接入。
metadata_jsonSTRING从 Execute 的 metadata_json 输出接入。
columnsInt网格列数。默认 4。
cell_widthInt网格每格宽度（像素）。默认 256。不同分辨率图片会自动缩放并居中。
max_grid_dimensionInt网格最大宽/高，超出自动缩放。默认 4096。
label_font_sizeInt标注字体大小。默认 16。自动 fallback：Arial → 微软雅黑 → DejaVu。
label_formatString标注格式模板。变量：{prompt} {lora_info} {cfg:.1f} {seed} {steps} {sampler} {scheduler} {index}
默认：C:{cfg:.1f} L:{lora_info}
filename_prefixString输出子目录。支持 %date:yyyy-MM-dd% 和 %time:HH-mm-ss% 占位符。
save_individualBool是否保存单张图。文件名含完整参数：{prompt}_lora0.50_cfg7.0_seed42_00001.png
save_gridBool是否保存标注网格对比图。
save_metadata_csvBool是否导出 CSV 元数据表（含所有参数列 + 文件名），方便后续打分筛选。
💡 CSV 用法：用 Excel / Numbers 打开，加一列"评分"，对每张图打分 1-5，筛选 4 分以上的看共同参数 → 你就知道最佳 CFG/Seed/Steps 区间了。

三、D站 Pipeline 工作流（推荐）

AnimaDex
Browser → Danbooru
Browser → AnimaPrompt
Converter → CLIPText
Encode → BatchDebug
Execute → Grid
Save

↑ 选好 D站 artist + 图片 → Tag 自动反推 → AnimaPromptConverter 拼装 → conditioning 直通 Execute。
不需要手写 prompt。只改 BatchDebugConfig 里的 LoRA 和 CFG 范围，点 Queue 即可。

四、快速上手

  1. 启动 ComfyUI（确认日志有 loaded 3 nodes）
  2. 拖入 d站_loRA_对比_workflow.json
  3. 在 AnimaDexBrowser 中选一个 artist（如 piromizu）
  4. 在 DanbooruBrowser 中选一张参考图
  5. 在 BatchDebugConfig 的 lora_sweep 中填你要测的 LoRA：
    anima线稿画风.safetensors 0.3 1.0 3
    anima美学增强，高清.safetensors 0.5 1.5 2
  6. 设 CFG 范围（如 3→9 分 4 步）
  7. 点 Queue，等终端出进度日志
  8. 到 ComfyUI/output/batch_debug/d站_loRA对比/ 看结果
